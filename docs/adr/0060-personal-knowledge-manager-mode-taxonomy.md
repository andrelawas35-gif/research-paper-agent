# Personal Knowledge Manager Mode Taxonomy

The Personal Knowledge Manager will use job-based Agent Modes rather than the older paper-centric mode set. Active modes are Capture, Retrieve, Source, Synthesis, Builder, Grill, Tutor, Reflect, Relationship, Taste, Review, Writing, Artifact, and Admin; deferred modes are Planning, Decision, and Briefing.

This taxonomy reflects the broader product shape: paper reading is one evidence-first capability inside a personal knowledge system, not the whole identity. Reader Mode remains an alias for Source Mode, and Profile Mode is absorbed into Reflect Mode because the user model now spans preferences, Candidate Signals, Tutor Progress, Concept Graph weights, notes, and relationship boundaries.

Mode routing should use one primary mode and at most one supporting mode, such as Synthesis supported by Retrieve or Tutor supported by Source. Modes are inferred silently by default and named only when doing so clarifies behavior, resolves ambiguity, or explains a mode switch.
