# Reminders Use a SQLite-Backed Idempotent Dispatcher

The single production VM runs one SQLite-backed background dispatcher for scheduled reminders and check-ins. It atomically claims due jobs, survives process restarts, applies each schedule's timezone behavior and configurable quiet hours, and uses idempotency keys to prevent duplicate delivery. Before delivery it rechecks permission, relevance, and current record state; sensitive wording is generated only at delivery time. Retries are bounded and visible, downtime recovery coalesces or expires stale reminders instead of releasing a burst, and delivery, seen, and acted outcomes are recorded separately. This avoids introducing Redis, Celery, Kafka, or a second scheduling service before the workload requires distributed coordination.
