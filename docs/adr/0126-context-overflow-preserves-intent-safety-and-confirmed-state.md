# Context Overflow Preserves Intent, Safety, and Confirmed State

When relevant material exceeds a Context Budget, assembly preserves the current message and explicit task first, followed by active safety state and non-overridable guardrails, relevant confirmed personal rules and values, active Work Object or Regulation Session state, direct evidence for the question, recent accepted decisions and commitments, clearly labelled provisional patterns, and finally general historical conversation summaries. Lower-priority material is omitted or compressed first. Compression may not collapse conflicting records into a falsely certain personal fact, and the model receives an explicit limitation signal when potentially relevant context was excluded. This makes overflow a deterministic application decision rather than an opaque model-side truncation.
