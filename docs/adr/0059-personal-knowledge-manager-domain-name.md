# Personal Knowledge Manager Domain Name

The product/domain name is now Personal Knowledge Manager because the agent manages papers, Personal Notes, relationship context, ideas, tutor progress, and the User Model rather than only research papers. The existing `research_paper_agent` package, service names, and deployment paths remain unchanged until a separate mechanical rename migration, avoiding import and deployment churn while the domain model settles.

Historical ADRs may continue to say "research agent"; they record the language and scope at the time they were written. Living docs, instructions, and future ADRs should use Personal Knowledge Manager, with old names updated only when they are part of current user-facing behavior or a planned mechanical rename.
