# Memory Review Is Batched and Non-Coercive

Most Memory Candidates do not interrupt an active conversation. The UI shows a quiet review-later indicator and places candidates in a Memory Inbox within the Data & Privacy Center, where review is offered in batches of roughly five. Immediate confirmation is reserved for a proposed record that would materially affect the current response. Each proposal uses plain language and exposes its source excerpt, date, confidence, domain, and conflicts, with accept, edit, defer, decline, and do-not-suggest-again actions. Safe candidates may be grouped for quick acceptance while uncertain ones remain individually inspectable. Weak unreviewed candidates expire automatically, and the interface uses no streaks, guilt language, or escalating badge pressure. This supports cumulative personalization without converting memory governance into a focus-breaking obligation.
