# Cloud Memory Processing Is Minimized and Consented Per Batch

Every Historical Backfill separately discloses local preprocessing and cloud model processing, and consent applies only to that selected batch. Local code removes connector metadata, unrelated messages, secrets, credentials, and unnecessary identifiers before sending only selected excerpts rather than complete archives. Regulation and intimate relationship material requires a separate explicit batch opt-in, the Owner can inspect a sample payload, and a local-only extraction option remains available with its quality trade-off stated. Provider requests use API settings that disable training and select the shortest supported retention controls; raw requests and responses do not enter ordinary logs; and derived candidates retain necessary provenance without preserving unnecessary prompt copies. Deep personalization therefore does not grant standing permission to treat conversation history as an unrestricted cloud dataset.
