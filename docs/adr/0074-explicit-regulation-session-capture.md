# Regulation Data Requires an Explicit Session

Durable regulation data is created only after the user explicitly enters Regulation Mode or starts a check-in; ordinary conversation does not silently become regulation history. Within a durable Regulation Session the agent may automatically structure the interaction into a compact Regulation Record, while a Private Check-In remains ephemeral and highly intimate narrative details require explicit retention. This trades some passive pattern-detection coverage for meaningful consent, lower sensitivity exposure, and a less noisy model of the user.
