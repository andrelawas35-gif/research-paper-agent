# Production Stack Favors Local-First Replaceable Components

The first production version uses Python 3.12, the existing Google ADK agent, FastAPI, and Pydantic on the backend, with a React, TypeScript, Vite PWA built from Tailwind CSS and accessible headless components. SQLite in WAL mode remains the primary database, with separate databases or enforceable store boundaries for operations, general PKM data, and sensitive Regulation data; retrieval uses FTS5 plus a replaceable local embedding index, and background work uses the SQLite-backed dispatcher. Separate systemd services run behind a private Tailscale network on the single Oracle VM. Sensitive records use application-level authenticated encryption with keys kept off the VM, recovery uses encrypted versioned off-VM backups with tested restores, and observability excludes sensitive content by default. Explicit repository, retrieval-index, queue, model-provider, and delivery adapters preserve migration paths to PostgreSQL, a dedicated vector database, distributed workers, or another model stack only when measured production needs justify them.
